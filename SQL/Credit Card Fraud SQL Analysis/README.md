# Credit Card Fraud SQL Analysis

## Project Overview

This project demonstrates SQL-based exploratory data analysis using a credit card fraud detection dataset from Kaggle. SQL queries were written using DuckDB within a Kaggle Notebook to analyze transaction patterns and generate fraud-related insights.

## Dataset

Credit Card Fraud Detection (Kaggle)

## Tools Used

- SQL
- DuckDB
- Kaggle Notebook
- GitHub

## SQL Concepts Demonstrated

- SELECT
- WHERE
- GROUP BY
- HAVING
- ORDER BY
- Aggregate Functions
- CASE WHEN
- Subqueries
- Window Functions

## Business Questions

- How many transactions are fraudulent?
- What is the fraud percentage?
- What is the average transaction amount?
- Which are the highest-value transactions?
- How many high-value transactions are there?
- What is the total value of fraudulent transactions?

## Key Insights

- Fraudulent transactions represent only a small portion of the dataset.
- Genuine transactions account for the majority of all transactions.
- High-value transactions are not always fraudulent.
- Most transactions are low in value.
- SQL can efficiently summarize and identify transaction patterns.
